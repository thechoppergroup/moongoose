## Project Overview

**Moongoose** is an SVG icon library for the Workstorm collaboration platform, published as `@thechoppergroup/moongoose` on npm.

### Key Facts

- **Icons**: 422 SVG icons organized by category
- **Framework**: Vue 3.5.0 (native, no compatibility layer)
- **Build Tool**: Webpack 5
- **Package Format**: UMD (CommonJS, AMD, and browser global support)
- **Status**: Phase 5.5 (stabilization post Vue 2→3 migration)
- **Consumers**: moonscape (direct), moonshine (indirect via moonscape)

### Icon Categories

Icons are organized by function:
- Navigation (menu, drawer, back, forward, etc.)
- Common (search, settings, user, folder, etc.)
- Document actions (save, download, upload, print, etc.)
- Social and messaging
- Notifications and alerts
- UI controls and form elements
- Status indicators

Each icon is defined as an SVG file in `src/icons/` and referenced by a kebab-case ID.

## Build Commands

All commands run from the `moongoose/` root directory.

### Primary Commands

```bash
npm install
# Install dependencies. Note: This overwrites npm link symlinks.
# After npm install, re-link into moonscape and moonshine.

npm run build
# Production build. Outputs UMD bundle to dist/moongoose.js
# Vue is externalized (not included in dist)
# Use this before npm link

npm run watch
# Continuous build on file changes. Good for development.
# Watch terminal for compile errors.

npm run serve
# Start dev server with HMR (hot module replacement)
# Serves preview app at http://localhost:8080
# Good for testing icons visually during development
```

### Secondary Commands

```bash
npm run release
# Build + version bump + publish to npm
# Only run this when releasing to npm registry
# Requires npm credentials

npm test
```

## Key Files Explained

### src/moongoose.vue

**Core icon component**. Renders a single SVG icon.

```vue
<template>
  <svg :class="['moongoose-icon', iconSize]">
    <use :xlink:href="`#${icon}`" />
  </svg>
</template>

<script>
export default {
  name: 'Moongoose',
  props: {
    icon: { type: String, required: true },
    size: { type: String, default: '24px' }
  },
  // ... rest of component
}
</script>
```

**Usage**:
```vue
<moongoose icon="menu" size="32px" />
```

### src/wrapper.js

**Plugin wrapper** for Vue 3. Registers the Moongoose component globally.

**Current Status**: NEEDS UPDATE
- Currently uses Vue 2 pattern: `install(Vue, options)`
- Should use Vue 3 pattern: `install(app, options)`

**See**: VUE3_DEVELOPER_GUIDE.md → "Vue 3 Plugin Pattern"

### src/preview.js

**Preview app entry point**. Renders the interactive documentation.

**Current Status**: NEEDS UPDATE
- Currently uses Vue 2: `new Vue({ el: '#app' })`
- Should use Vue 3: `createApp().mount('#app')`

**See**: VUE3_DEVELOPER_GUIDE.md → "App Creation Pattern"

### webpack.config.js

**Main webpack configuration**.

**Critical Configuration**: Vue is externalized
```javascript
externals: {
  vue: { commonjs: 'vue', commonjs2: 'vue', amd: 'vue', root: 'Vue' }
}
```

This prevents Vue duplication in consuming apps (moonscape, moonshine).

**Do NOT change this externals config** unless you have a specific reason.

### package.json

**Dependencies and build scripts**.

**Important fields**:
- `"engines": { "node": ">=22" }` — Enforces Node 22 LTS
- `"dependencies": { "vue": "^3.5.0" }` — Vue 3 (only for dev)
- `@vue/compat` — SHOULD BE REMOVED (unused)

## Dependency Chain

Moongoose is the base of a three-project dependency chain:

```
moongoose (SVG icons + core component)
    ↓
moonscape (UI component library built with moongoose icons)
    ↓
moonshine (Frontend app using moonscape components)
```

### Important Implications

1. **Changes to moongoose ripple downstream**: After modifying icons or the core component, must rebuild moonscape, then moonshine
2. **npm link is required for development**: Pre-compiled versions on npm use the previous build
3. **npm install breaks links**: After `npm install`, must re-link symlinks into moonscape and moonshine

### npm link Procedure

After making changes to moongoose:

```bash
# 1. From moongoose/ root
npm run build
npm link

# 2. From moonscape/ root
npm link @thechoppergroup/moongoose
npm run build

# 3. From moonshine/ root (if using updated moonscape)
npm link @thechoppergroup/moonscape
npm run build

# 4. Test in moonshine development server
npm run dev
```

**After npm install anywhere, re-run the npm link procedure.**

## Common Tasks

### Adding a New Icon

```bash
# 1. Create SVG file
cp new-icon.svg src/icons/

# 2. Update icon metadata
# Edit src/docs/meta.json, add entry:
{
  "name": "new-icon",

  "tags": ["menu", "sidebar"]

npm link
cd ../moonscape && npm link @thechoppergroup/moongoose
npm run build
```

### Modifying a Component

```bash
# 1. Edit component file (e.g., src/components/editor.vue)

# 2. Rebuild
npm run build

# 3. Test visually
npm run serve
# OR in consuming app with npm link:
cd ../moonscape && npm run dev

# 4. Test in full chain (moonscape → moonshine)
npm run build  # moonscape
cd ../moonshine && npm run dev
```

### Debugging

```bash
# Check compilation errors
npm run watch
# Keep terminal open, watch for errors as you edit

# Check bundle size
du -h dist/moongoose.js
# Should be 50-80 KB (uncompressed), 10-20 KB (gzipped)

# Verify Vue is externalized (not in bundle)
grep -o "require.*vue" dist/moongoose.js | head -5
# Should show references to external 'vue' module, not bundled code
```

## Vue 3 Specifics for This Project

### Vue 3 Features Used

- **Options API**: All components use Options API (no Composition API yet)
- **Templates**: All components use templates (no render functions)
- **Reactive Properties**: data(), computed, watch
- **Lifecycle**: created(), mounted() (Vue 2 hooks still work)
- **Props & Events**: Standard with validation and emits declarations

### Vue 3 Features NOT Used

- Composition API (available but not used)
- `<script setup>` syntax (available but not used)
- TypeScript (not used)
- Server-side rendering (not tested)

### Webpack Vue Externalization

Vue is intentionally NOT bundled with Moongoose:

```javascript
// webpack.config.js
externals: {
  vue: {
    commonjs: 'vue',
    commonjs2: 'vue',
    amd: 'vue',
    root: 'Vue'
  }
}
```

**Why**: Consuming apps (moonscape, moonshine) provide Vue 3 themselves. Prevents duplicate Vue in final bundle.

**Implication**: Moongoose dist assumes Vue 3 is available in the consuming app.

## Code Style

### File Naming

- **Vue components**: PascalCase (exported), stored as kebab-case.vue
  - Example: `EditorComponent` exported from `editor.vue`
- **JavaScript files**: kebab-case
  - Example: `preview.js`, `wrapper.js`
- **Icon files**: kebab-case with .svg extension
  - Example: `menu.svg`, `user-profile.svg`

### Component Organization

```javascript
// Order of sections in components:
export default {
  name: 'ComponentName',           // 1. Name
  props: { /* ... */ },            // 2. Props (with validation)
  emits: ['event-name'],           // 3. Emitted events
  data() { /* ... */ },            // 4. Data
  computed: { /* ... */ },         // 5. Computed properties
  watch: { /* ... */ },            // 6. Watchers
  created() { /* ... */ },         // 7. Lifecycle hooks
  mounted() { /* ... */ },
  methods: { /* ... */ }           // 8. Methods
}
```

### Commenting Style

Use clear, concise comments explaining the "why":

```javascript
// ✓ Good: Explains the purpose
watch: {
  searchQuery(newVal) {
    // Debounce API calls to avoid rate limiting
    clearTimeout(this.searchTimeout)
    this.searchTimeout = setTimeout(() => this.fetchResults(newVal), 300)
  }
}

// ❌ Bad: Just restates the code
watch: {
  searchQuery(newVal) {
    // Watch the search query
    // ...
  }
}
```

## Troubleshooting

### Icons don't appear in moonscape

**Likely cause**: npm link symlink was overwritten by npm install

**Fix**:
```bash
cd moongoose && npm run build && npm link
cd ../moonscape && npm link @thechoppergroup/moongoose && npm run build
```

### Webpack compilation error

**Check**: Are you using the correct Node version?
```bash
node --version  # Should be 22+
```

**Check**: Is @vue/compat still in package.json?
```bash
npm uninstall @vue/compat
npm install
npm run build
```

### Preview app shows blank page

**Likely cause**: preview.js uses Vue 2 pattern

**Status**: This is a known issue, needs the update documented in VUE3_DEVELOPER_GUIDE.md

### Bundle size too large

**Check**: Is Vue included in the bundle?
```bash
grep -i "vue.*version" dist/moongoose.js | head -3
# Should show external reference, not version number
```

**Check**: Are there duplicate icons?
```bash
ls -la src/icons/ | wc -l  # Should be ~422
```
